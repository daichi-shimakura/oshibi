# oshibi - お笑いライブスケジュール管理アプリ

**「芸人は推されやすく、ファンは推しやすく」**

お笑い芸人とファンのためのライブスケジュール管理プラットフォームです。  
芸人が自分のライブ情報を発信し、ファンが推しの出演予定をまとめて確認できます。

🔗 **https://oshibi.onrender.com**

> [!NOTE]
> 初回アクセス時、サーバー起動に30秒〜1分ほどかかる場合があります。

---

## なぜ作ったのか

お笑い芸人として活動する中で、ライブ情報が X・Instagram・Note などに散らばっていて、ファンが追いきれないという課題を日常的に感じていました。

既存サービス（Hashigo 等）は「ライブ情報のデータベース」ですが、oshibi は **「芸人が主役」** のアプリです。同じライブでも芸人ごとに違う情報が見える——芸人 A にとっては「新ネタ 3 本おろすライブ」、芸人 B にとっては「大好きな先輩との共演ライブ」。この **ライブ情報の 2 層構造**（共通ライブ情報 ＋ 芸人別ライブ情報）が oshibi の核です。

## 主な機能

### 閲覧（ログイン不要）
- **ライブ一覧・検索** — キーワード・日付・都道府県・ライブ種別・価格帯・時間帯で絞り込み
- **ライブ詳細** — 公演情報・出演者・会場情報を一画面で確認
- **芸人一覧・検索** — 芸名・ユニット種別・事務所で検索
- **芸人詳細** — プロフィール・SNS リンク・出演ライブ一覧
- **芸人別ライブページ** — その芸人にとってのそのライブの情報（本予定/仮予定・ネタ情報・コメント）

### 登録・編集（要ログイン）
- **ユーザー登録** — ファン / ライブ関係者の 2 種類のアカウント
- **ライブ情報の CRUD** — ライブ関係者が共通情報を登録・編集・削除
- **出演者登録** — 芸人を非同期検索して出演者として追加（出番順の並べ替え対応）
- **芸人別ライブ情報の編集** — 出演者本人のみが本予定/仮予定・ネタ情報・コメントを編集
- **プロフィール編集・アカウント設定**

## 設計のポイント

### ライブ情報の 2 層構造

```
lives（共通ライブ情報：ライブ名・日時・会場・料金等）
  └── live_performers（芸人別ライブ情報）
        ├── 本予定 / 仮予定ステータス
        ├── ネタ本数・ネタ種類
        └── ファンへのコメント
```

単なる中間テーブルではなく、芸人ごとの「このライブに対する情報」を持たせることで、同じライブが芸人ごとに違う見え方になります。

### 認可の 2 層設計

1. **SecurityConfig（ロールレベル）** — URL パターンでファン / ライブ関係者のアクセスを制御
2. **Service 層（本人確認）** — ライブ編集は登録者本人のみ、芸人別情報は出演者本人のみ

Spring Security だけでは「ライブ関係者ならだれでも他人のライブを編集できる」状態になるため、Service 層で作成者 ID の一致を検証しています。

### Specification パターンによる検索

ライブ一覧の複合検索（キーワード・日付・都道府県・種別・価格帯・時間帯）に JPA Specification パターンを採用。検索条件の組み合わせが多いため、動的にクエリを構築できるこのパターンが適しています。

### @EntityGraph による N+1 問題の解消

ライブ一覧取得時に出演者情報を JOIN FETCH することで、N+1 クエリを回避しています。

### 出演者登録の非同期検索

ライブ登録画面での芸人追加は、`@RestController` + fetch API による非同期通信で実装。画面遷移なしで芸人を検索・追加できます。

## ER 図

```
users ──── accounts ──── comedian_profiles
                │
                │ (created_by)
                ▼
venues ──── lives ──── live_performers ──── comedian_profiles
```

| テーブル | 概要 |
|---|---|
| `users` | ログイン情報（メール・パスワード） |
| `accounts` | アカウント情報（種別：FAN / LIVE_STAFF・表示名・SNS リンク等） |
| `comedian_profiles` | 芸人プロフィール（ユニット種別・事務所・メンバー名） |
| `venues` | 会場マスタ（会場名・住所・最寄り駅・Google Maps URL） |
| `lives` | ライブ情報（日時・料金・配信情報等） |
| `live_performers` | 芸人別ライブ情報（ステータス・ネタ情報・コメント） |

## ローカル開発環境のセットアップ

### 前提条件
- Java 17
- PostgreSQL
- Gradle

### 手順

```bash
# 1. リポジトリをクローン
git clone https://github.com/daichi-shimakura/oshibi.git
cd oshibi

# 2. PostgreSQL にデータベースを作成
createdb oshibi

# 3. application.properties の DB 接続情報を環境に合わせて編集

# 4. アプリケーションを起動（Flyway が自動でテーブルを作成します）
./gradlew bootRun

# 5. ブラウザでアクセス
# http://localhost:8080
```
